1. Description of this project

This project involves assembling an HDR image from a series of photographs taken at different exposures. HDR images capture a wider dynamic range and maintain a linear relationship with physical irradiance, making them useful for graphics and vision applications.

3. Image alignment

We implemented a brute-force version of the Medium Threshold Bitmap (MTB) algorithm for image alignment.

The process begins by converting each input image to grayscale and determining its median intensity value. This median serves as a threshold to create a binary bitmap, where pixels above the threshold are set to 111, and those below are set to 000.

To align the images, we select the one with the middle exposure as the reference and shift the remaining images to minimize their differences with the reference image.

To reduce threshold noise, we also added the mask that excludes pixels with grayscale values near the threshold. Specifically, we ignore pixels within the range:

threshold±2\mathrm{threshold} \pm 2 threshold±2

This approach yields the best results when working with 8-bit grayscale images.

Since we employ a brute-force search for the best alignment, we limit the maximum shift to 10 pixels. However, given that the images were captured using a tripod, only minor shifts were necessary, and a maximum shift of 10 proved to be more than sufficient.

Regarding implementation details, when shifting images, we fill in the blank areas using nearby pixels as a reasonable approximation. However, this filling is only applied during the difference calculation and does not affect the final aligned images. Once the optimal shifts are determined, we crop the images to remove any artificially introduced pixels. Given that the maximum shift is limited to 10 pixels, the cropping ensures that at most 10 pixels are removed from each side of the image.

5. Tone mapping

Log Min-max Scaling

We first implemented the simple Min-Max normalization.

We converted the values of each channel to log space and simply scale it using min-max scaling. We use PR1 and PR99 of the values as the lower bound (lblblb) and upper bound (ububub) to counter outliers.

Lout=Lin−lbub−lb L_\text{out} = \frac{L_\text{in} - lb}{ub - lb} L​out​​=​ub−lb​​L​in​​−lb​​

This simple approach is surprisingly already good enough for us to see the details of both the dark side (the statue) and the bright side (outside of the window).

Reinhard Method with Split Channel

Next, we implemented the Reinhard tonemapping method.

For each channel, we first compute the log-average luminance:

Lavg=exp(log(δ+Lin)) L_\text{avg} = \exp\left(\log(\delta + L_\text{in})\right) L​avg​​=exp(log(δ+L​in​​))

where δ\deltaδ is set to 10−510^{-5}10​−5​​ to avoid logarithm singularities.

Next, we normalize the input luminance based on this value:

Lm=Lin⋅αLavg L_\text{m} = L_\text{in} \cdot \frac{\alpha}{L_\text{avg}} L​m​​=L​in​​⋅​L​avg​​​​α​​

Here, α\alphaα is a brightness scaling parameter, which we set to 0.450.450.45.

Finally, we apply the Reinhard remapping function to compress the luminance into the [0,1][0, 1][0,1] range.

Lout=Lm1+Lm⋅1+LmLwhite2 L_\text{out} = \frac{L_\text{m}}{1 + L_\text{m}} \cdot \frac{1 + L_\text{m}}{L_\text{white}^2} L​out​​=​1+L​m​​​​L​m​​​​⋅​L​white​2​​​​1+L​m​​​​

where LwhiteL_\text{white}L​white​​ controls the maximum distinguishable luminance. Since our scene does not contain extremely bright light sources, we set Lwhite=30L_\text{white} = 30L​white​​=30, effectively disabling this feature.

In the tone-mapped image, we observe that both bright and dark regions retain visible details. However, compared to the previous method, this approach introduces a yellowish tint, producing a warmer color tone. This effect gives the image a subtle vintage aesthetic.

Reinhard Method with Grayscale Image

We then realized that the Reinhard method should be applied to the luminance of the entire image, rather than processing each channel separately. To address this, we implemented an improved version.

First, we convert the image to grayscale to extract the global luminance and apply the same tone mapping function to scale it into the [0,1][0,1][0,1] range. We then use this mapped luminance to adjust the color channels proportionally:

Cout=Cin⋅LoutLin C_\text{out} = C_\text{in} \cdot \frac{L_\text{out}}{L_\text{in}} C​out​​=C​in​​⋅​L​in​​​​L​out​​​​

where CinC_\text{in}C​in​​ represents the original color channel values, and CoutC_\text{out}C​out​​is the adjusted output.

We observed that this approach tends to increase saturation, making the image appear overly vivid. To counter this, we convert the result to HSV color space and reduce the saturation by a factor of 0.80.80.8.

Compared to the previous methods, this approach not only preserves details in both bright and dark regions but also maintains a more natural color balance. As a result, this became our preferred tone-mapping method.
